# 修正仕様書：ITD脈動閾値実験プログラムのDSPパイプライン最適化

## 1. 目的
`phase2_stimulus.py` 内の信号生成プロセスを改修する。
1. マスカーノイズを「Frozen Noise」から「Running Noise」へ変更し、位相キャンセルのアーティファクトを排除する。
2. マスカーのレベル定義を「全体レベル（Overall Level）」から「スペクトルレベル（dB/Hz）」に基づいた計算へとアップグレードする。
（※ITD付与時のスライス処理およびCosine Rampの順序は現在完璧なため、変更しないこと）

## 2. 修正指示

### Task 1: マスカーの Running Noise（ランダム化）化
**対象:** `build_alternating_stimulus` 関数
* **現状:** `m_sig` がループの外部で1度だけ生成され、交番刺激（M-T-M-T...）の全てのM区間で使い回されている。
* **修正:** M（マスカー）のセグメントを配置するタイミング（`is_masker = True` のブロック内）で、毎回 `generate_bandpass_noise` を呼び出し、各バーストが独立したランダム波形になるように書き換えること。

### Task 2: dB/Hz (Spectrum Level) ベースのマスカーレベル計算
**対象:** `build_alternating_stimulus` の引数および計算ロジック
* `masker_level_db` という引数を `masker_spectrum_level_db` (単位: dB/Hz) という名前に変更（または意味論を更新）する。
* マスカー生成前に、以下の計算式を用いて「全体の要求レベル（Overall RMS Level）」を計算する。
  1. `f_low = center_freq / np.sqrt(2)`
  2. `f_high = center_freq * np.sqrt(2)`
  3. `bandwidth = f_high - f_low`
  4. `overall_level_db = masker_spectrum_level_db + 10 * np.log10(bandwidth)`
* `generate_bandpass_noise` を呼び出す際は、この計算された `overall_level_db` を渡すこと。

## 3. 期待される出力
* `build_alternating_stimulus` 内で `m_sig` の使い回しが完全に排除されていること。
* マスカーの生成レベルが、帯域幅（Bandwidth）のエネルギー総量を考慮したスペクトルレベル（dB/Hz）から逆算されていること。
